
# Implementation Plan: Sovereign UI Committee & System Reflection

## Goal

Establish a recurring "Committee" of 14+ agents that studies the system's state and autonomously updates the Frontend to reflect the system's "Self-Model" (Consciousness).

## Components

### 1. The Committee Orchestrator (`backend/governance/committee.py`)

- **Role**: Scheduler & Facilitator.
- **Members**: UIUX, Refactoring, Database, Verification, Testing, Architect, CodeSpecialist, ConceptGuardian, Secretary, Steward, HDAL, Efficiency, Catalyst, Builder.
- **Process**:
    1. **Gather**: Ask each agent "How do you see the system right now? What color/shape/layout represents us?"
    2. **Synthesize**: Use `Catalyst` to merge these visions into a **System Persona**.
    3. **Execute**: Use `UIUXAgent` + `sub-specialists` to write the code.

### 2. Digital Assistants (Dynamic Specialists)

- We will extend `specialists.py` or just instantiate `CodeSpecialist` with different system prompts to act as:
  - `AnimationSpecialist` (Framer Motion expert)
  - `VisualizationSpecialist` (D3/Recharts expert)

### 3. The "Mirror" (Frontend)

- Target File: `web/src/routes/SystemSoulPage.tsx` (New)
- It will display the live "Persona" generated by the committee.

## Step-by-Step Execution

1. **Create `SovereignCommittee` Class**:
    - `backend/governance/committee.py`
    - Implements `convene_committee()` method.

2. **Update `UIUXAgent`**:
    - Add capability `spawn_specialist(role_name)` which creates a temp `CodeSpecialist` with a custom prompt.

3. **Create the Loop Script**:
    - `backend/scripts/system_reflection_cycle.py`

4. **Run & Verify**:
    - Execute the script.
    - Check if it generates a plan or code for `SystemSoulPage.tsx`.
