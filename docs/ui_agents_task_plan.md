# خطة تنفيذ برمجية تفصيلية لتطوير واجهة مراقبة وكلاء ICGL

---

## 1. إعداد هيكل المشروع

- إنشاء مجلد جديد للواجهة (مثلاً: web/src/routes/AgentsFlowPage)
- تجهيز ملفات المكونات الأساسية (HeaderBar, WorkflowBoard, AgentCard, EventLog, StatusIndicators, IdeaSummary, AgentDetailsModal)

---

## 2. بناء المكونات البرمجية

### أ. HeaderBar

- عرض اسم النظام، زر العودة، إعدادات

### ب. WorkflowBoard

- رسم بياني أو Timeline يوضح مراحل معالجة الفكرة
- عرض اسم الوكيل المسؤول وحالة التنفيذ لكل مرحلة

### ج. AgentCard

- عرض بيانات كل وكيل (اسم، دور، حالة، مهمة، توصية)
- زر "تفاصيل أكثر" لعرض سجل الأحداث المرتبط بالوكيل

### د. EventLog

- قائمة زمنية للأحداث (تحليل، توصية، تضارب، تنفيذ)
- ربط كل حدث بالوكيل المنفذ

### هـ. StatusIndicators

- رموز أو شريط حالة يوضح حالة النظام العامة

### و. IdeaSummary

- عرض الفكرة المدخلة وما فهمه النظام منها

### ز. AgentDetailsModal

- نافذة منبثقة تعرض تفاصيل أعمق لكل وكيل أو مرحلة

---

## 3. ربط المكونات بواجهات API

- إعداد دوال جلب البيانات من API (agents, analysis, events, status, idea-summary)
- ربط كل مكون بالبيانات الحية المطلوبة
- معالجة حالات التحميل والأخطاء

---

## 4. اختبار التكامل وتجربة المستخدم

- اختبار تدفق البيانات بين المكونات
- التأكد من وضوح العرض وعدم ازدحام العناصر
- تجربة الواجهة على مختلف الأجهزة والمتصفحات

---

## 5. تحسينات وتوسعات مستقبلية

- دعم تخصيص العرض (تغيير الألوان، ترتيب المكونات)
- إضافة دعم اللغة الإنجليزية
- إمكانية تصدير سجل الأحداث أو ملخص الفكرة
- ربط الواجهة الجديدة بالواجهة الحالية عبر زر أو رابط واضح

---

## 6. التوثيق والتسليم

- توثيق جميع المكونات البرمجية وواجهات API
- تحديث دليل الاستخدام
- تسليم المشروع مع جميع الملفات والتقارير المساندة

---

هذه الخطة التفصيلية ستكون المرجع لكل خطوة في تطوير الواجهة الجديدة.
