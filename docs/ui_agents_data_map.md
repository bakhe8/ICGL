# خريطة بيانات وكلاء ICGL للواجهة الجديدة

## 1. قائمة الوكلاء وأدوارهم

- ArchitectAgent: تحليل الهيكلية والتصميم
- PolicyAgent: تطبيق السياسات والحوكمة
- FailureAgent: كشف سيناريوهات الفشل والمخاطر
- SentinelAgent: مراقبة المخاطر والانحرافات اللحظية
- GuardianAgent: حماية المفاهيم الأساسية للنظام
- BuilderAgent: توليد الكود البرمجي
- EngineerAgent: تنفيذ ونشر التغييرات البرمجية
- MediatorAgent: تنسيق التعاون وحل النزاعات بين الوكلاء
- DocumentationAgent: توليد وتحسين الوثائق البرمجية
- Specialists: تحليل تخصصي في مجالات محددة
- AgentRegistry: إدارة دورة حياة الوكلاء وتنسيقهم
- Agent Base: الأساس البرمجي لجميع الوكلاء
- CapabilityChecker: فحص القدرات وتحديد الفجوات بين الوكلاء
- RefactoringAgent: تحسين وإعادة هيكلة الكود
- VerificationAgent: التحقق العميق من الكود
- TestingAgent: توليد وتنفيذ الاختبارات البرمجية

## 2. البيانات المطلوبة لكل وكيل في الواجهة

- اسم الوكيل
- الدور الأساسي
- الحالة (نشط/بانتظار/تدخل لحل تضارب)
- المهمة الحالية أو الأخيرة
- نتيجة التحليل أو التوصية
- حالة التنفيذ (نجاح/تضارب/تحذير)
- سجل الأحداث المرتبطة بالوكيل

## 3. نقاط التكامل بين الوكلاء

- من يزود من بالمعلومات؟
- من يعتمد على نتائج من؟
- من يتدخل لحل النزاعات؟
- كيف يتم تمرير البيانات بين الوكلاء؟

## 4. متطلبات واجهة المستخدم من حيث البيانات

- عرض تدفق العمل خطوة بخطوة
- بطاقات لكل وكيل نشط
- سجل الأحداث الزمني
- مؤشرات حالة النظام
- ملخص الفكرة المدخلة وما فهمه النظام منها
- تفاصيل أعمق عند الحاجة (زر "تفاصيل أكثر")
