# متطلبات التطوير البرمجية لواجهة مراقبة وكلاء ICGL

---

## 1. المكونات البرمجية المطلوبة

- شريط علوي (HeaderBar)
- لوحة تدفق العمل (WorkflowBoard)
- بطاقة وكيل (AgentCard)
- سجل الأحداث (EventLog)
- مؤشرات الحالة (StatusIndicators)
- ملخص الفكرة (IdeaSummary)
- زر "تفاصيل أكثر" (AgentDetailsModal)

---

## 2. البيانات المطلوبة من الخلفية (Backend)

- قائمة الوكلاء النشطين وأدوارهم
- حالة كل وكيل (نشط/بانتظار/تدخل لحل تضارب)
- المهمة الحالية أو الأخيرة لكل وكيل
- نتيجة التحليل أو التوصية لكل وكيل
- سجل الأحداث الزمني (من استقبال الفكرة حتى التنفيذ)
- حالة النظام العامة (نجاح/تضارب/تحذير)
- ملخص الفكرة المدخلة وما فهمه النظام منها

---

## 3. واجهات API المطلوبة

- GET /api/system/agents — جلب قائمة الوكلاء وحالتهم
- GET /api/analysis/{adr_id} — جلب نتائج التحليل لكل فكرة
- GET /api/events — جلب سجل الأحداث الزمني (إن وجد)
- GET /api/status — جلب حالة النظام العامة
- GET /api/idea-summary/{adr_id} — جلب ملخص الفكرة وما فهمه النظام منها

---

## 4. متطلبات التصميم والتفاعل

- ألوان خلفية فاتحة، خطوط واضحة، مساحات واسعة بين العناصر
- عرض مختصر لكل وكيل مع إمكانية التوسع لرؤية التفاصيل
- عرض تدفق العمل بشكل بصري وتفاعلي
- سجل أحداث قابل للتمرير والاستعراض
- مؤشرات حالة واضحة وسهلة الفهم
- دعم اللغة العربية والإنجليزية
- توافق مع الأجهزة المكتبية والمحمولة

---

## 5. خطوات التنفيذ البرمجية

1. بناء مكونات الواجهة بشكل منفصل (React Components)
2. ربط المكونات بواجهات API لجلب البيانات الحية
3. اختبار التكامل بين المكونات والبيانات
4. تحسين تجربة المستخدم بناءً على الملاحظات
5. توثيق جميع المكونات والواجهات البرمجية

---

سيتم البناء على هذه المتطلبات في الخطوات التالية لتطوير الواجهة البرمجية.
