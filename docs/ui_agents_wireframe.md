# تصميم تخطيطي (Wireframe) لواجهة مراقبة وكلاء ICGL

---

## 1. الهيكل العام للواجهة

```text
┌───────────────────────────────────────────────┐
│ شريط علوي: اسم النظام | زر العودة | إعدادات │
├───────────────────────────────────────────────┤
│ لوحة تدفق العمل (Workflow Board)             │
│  [ استقبال الفكرة ] → [ تحليل الوكلاء ] → ... │
├───────────────────────────────────────────────┤
│ بطاقات الوكلاء (Agent Cards)                  │
│ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐│
│ │ Architect   │ │ Builder     │ │ Policy      ││
│ │ ...         │ │ ...         │ │ ...         ││
│ └─────────────┘ └─────────────┘ └─────────────┘│
├───────────────────────────────────────────────┤
│ سجل الأحداث (Event Log)                       │
│ [تحليل] [توصية] [تضارب] [تنفيذ] ...          │
├───────────────────────────────────────────────┤
│ مؤشرات الحالة (Status Indicators)             │
│ [نجاح] [تضارب] [تحذير]                        │
├───────────────────────────────────────────────┤
│ ملخص الفكرة (Idea Summary)                    │
│ [الفكرة المدخلة] [ما فهمه النظام]             │
└───────────────────────────────────────────────┘
```

---

## 2. وصف المكونات

### أ. شريط علوي

- اسم النظام (ICGL Cockpit)
- زر العودة للواجهة الرئيسية
- إعدادات (تغيير اللغة، تخصيص العرض)

### ب. لوحة تدفق العمل

- رسم بياني أو Timeline يوضح مراحل معالجة الفكرة خطوة بخطوة.
- كل مرحلة تعرض اسم الوكيل المسؤول وحالة التنفيذ.

### ج. بطاقات الوكلاء

- لكل وكيل بطاقة تعرض:
  - الاسم والدور
  - المهمة الحالية أو الأخيرة
  - نتيجة التحليل أو التوصية
  - حالة التنفيذ
  - زر "تفاصيل أكثر" لعرض سجل الأحداث المرتبط بالوكيل

### د. سجل الأحداث

- قائمة زمنية تظهر كل حدث تم في الخلفية.
- كل حدث مرتبط بالوكيل المنفذ، مع وصف مختصر ووقت التنفيذ.

### هـ. مؤشرات الحالة

- رموز أو شريط حالة يوضح حالة النظام العامة (نجاح، تضارب، تحذير).

### و. ملخص الفكرة

- مربع يعرض الفكرة المدخلة، وما فهمه النظام منها، وكيف تم توزيعها على الوكلاء.

---

## 3. تدفق البيانات بين المكونات

- استقبال الفكرة → توزيعها على الوكلاء → عرض نتائج كل وكيل في بطاقته → تسجيل كل حدث في سجل الأحداث → تحديث مؤشرات الحالة.

---

## 4. نقاط التفاعل مع المستخدم

- إدخال فكرة جديدة.
- التنقل بين تفاصيل كل وكيل.
- استعراض سجل الأحداث بالكامل.
- العودة للواجهة الرئيسية.
- تخصيص العرض أو الإعدادات.

---

هذا التصميم التخطيطي سيكون الأساس لتطوير الواجهة البرمجية في الخطوات التالية.
